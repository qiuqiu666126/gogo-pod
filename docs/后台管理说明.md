# POD 后台与管理端说明

> **当前阶段**：管理端以 **UI 原型** 为主，场景表单配置保存在浏览器 localStorage，**尚未对接 API**。

## 打开管理后台

```bash
npm install
npm run dev
# 或 npm run dev:admin
```

| 地址 | 说明 |
|------|------|
| http://localhost:5173/admin.html | 推荐 |
| http://localhost:5173/admin | 自动跳转 |

---

## 架构分工

| 模块 | 职责 |
|------|------|
| **AI 功能配置** | 仅：功能名称、说明、启用状态、model_id、API Key |
| **场景预设** | 每个功能 × 每个场景的前台表单控件 + 提示词 |

表单与提示词 **不再** 在 AI 功能配置里维护。

## 场景预设：动态表单配置

路径：**场景预设**

### 各功能场景一览

| 功能 | 场景 key | 说明 |
|------|----------|------|
| 印花图提取 | 默认 | 提取模式、透明底、分辨率、比例 |
| 一键抠图 | 默认 | 抠图模式、边缘处理 |
| 图裂变 | 默认 / 服装/纺织 / … | 多 Tab，模式子控件（见下例） |
| 文生图 | 默认 | 模式、比例、出图数量（提示词列表仍为前台专用 UI） |
| 转矢量图 | 默认 | 原图风格 |
| 侵权风险过滤 | 默认 | 深度/基础过滤 |
| 商品套图 | 印花处理 | 第二步印花处理参数（模板选择仍为向导 UI） |
| 视频生成 | model / product / wind | 三 Tab：模特动作、商品律动、风铃转动 |

首次加载或升级后，缺失的预设会自动合并进 localStorage（按预设 id 去重）。

### 操作步骤（图裂变 · 文字强化为例）

1. 功能选「图裂变」
2. 编辑场景「默认」（对应前台顶部 Tab）
3. 在「表单控件」中展开 **模式选择 → 文字强化 → 子控件**
4. 配置子控件：
   - 原图参考强度：滑条，0~1，默认 0.5
   - 创意发散强度：滑条，显示文字映射
   - 指定背景色：单选，随机/黑/白
5. 右侧 **前台预览** + **合成提示词预览** 实时更新
6. 保存后用户端图裂变 Modal 立即生效

### 控件类型（固定）

| 类型 | 用途 |
|------|------|
| 单选 radio | 模式选择、背景色等 |
| 下拉 select | 比例、商品类型 |
| 滑条 slider | 参考强度、发散强度 |
| 勾选 checkbox | 开关类选项 |
| 单行/多行文本 | 自定义 prompt 输入 |
| 数字按钮 | 出图数量 1/2/4/6/8 |

### 选项子控件

单选/下拉的 **每个选项** 可展开配置「子控件」——用户选中该选项时才显示。  
例如「模式选择 → 文字强化」下挂三个滑条/单选，「艺术设计」下挂另一套。

### 提示词合成规则

```
完整提示词 = 场景基础提示词
           + 各可见控件的 promptFragment（{{value}} 替换为用户值）
           + 选中选项的 promptFragment
```
