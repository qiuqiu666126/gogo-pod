# GOGO POD 管理后台 - 产品需求文档 (PRD)

## 1. 产品概述
**GOGO POD 管理后台** 是支撑“GOGO POD 前台用户端”正常运转的核心中枢系统。它主要负责平台的全局运营数据监控、AI 底层能力的配置与路由、平台级运营模板（工作流、套图、推荐案例）的维护，以及前台用户与系统底层设置的管理。

该系统确保了平台能灵活应对底层 AI 模型的切换，快速响应市场玩法上新，并实现商业化运营管控。

## 2. 目标角色
- **平台运营人员**：负责配置推荐案例、官方工作流模板、套图模板，管理前台用户账号。
- **AI 算法/研发工程师**：负责配置与调试 8 大 AI 功能的模型接口、提示词（Prompt）策略及限额规则。
- **系统管理员**：负责查看全站运行大盘数据，管理全局系统设置（如 OSS、网关 Token）。

## 3. 站点信息架构 (IA)

管理后台通过独立入口（`admin.html`）访问，主要包含以下 9 大功能页面：
1. **概览 (Dashboard)**：全局统计与预警大盘。
2. **AI 功能配置 (Feature Detail)**：前台核心作图/视频底层模型与提示词映射管理。
3. **场景预设 (Presets)**：AI 任务的场景化 Prompt 库管理。
4. **推荐案例 (Recommendations)**：前台首页展示的优质生成案例管理。
5. **工作流模版 (Workflow Templates)**：前台官方批量工作流编排与下发。
6. **套图模版 (Product Set Templates)**：真实商品样机与蒙版的入库管理。
7. **用户账号 (Users)**：前台登录用户的开设与权限管控。
8. **任务监控 (Tasks)**：全局 AI 任务排队与执行状态查表。
9. **系统设置 (Settings)**：环境配置与基础秘钥管理。

## 4. 核心功能说明

### 4.1 数据概览大盘 (Dashboard)
*对应代码：`DashboardPage.tsx`*
- **统计指标**：展示全站注册用户数、功能调用次数、预设模板数量等核心运营数据。
- **预警提醒**：实时检测底层 AI 配置（如 Provider API Key 缺失、限额耗尽），并向管理员提供可视化警报。

### 4.2 AI 功能引擎配置 (Feature Configuration)
*对应代码：`FeatureDetailPage.tsx`*
本模块是系统的核心引擎，用于控制前台 8 大 AI 功能（抠图、印花提取、裂变、文生图、套图、矢量、侵权、视频）的底层逻辑：
- **模型路由与通道 (Provider)**：可配置不同功能调用的底层模型及对应的 API Key。
- **提示词工程 (Prompt Management)**：支持配置“系统提示词 (System Prompt)”与“用户提示词 (User Prompt)”模板。
- **参数映射**：将前台用户的 UI 交互参数（如清晰度、比例）映射为底层大模型可识别的参数格式。
- **调用限额**：针对单账号或单功能设置调用频次上限。

### 4.3 场景预设管理 (Presets)
*对应代码：`PresetsPage.tsx`*
- 配合“AI 功能引擎”，为特定的 AI 功能（尤其是图裂变、文生图、视频生成）建立场景库。
- 管理员可预埋不同风格（如铁艺、挂钟、服饰）的 Scene Prompt 模板（CRUD），供前台用户一键调用。

### 4.4 平台模板运营管理
*对应代码：`RecommendationsPage.tsx`, `WorkflowTemplatesAdminPage.tsx`, `ProductSetTemplatesAdminPage.tsx`*
- **推荐案例 (Recommendations)**：配置前台首页“推荐玩法”模块展示的图文/视频卡片及相关标签。
- **官方工作流模版**：
  - 支持创建、编辑、排序和启停官方下发的工作流。
  - 内置与前台一致的**可视化工作流编排器 (WorkflowBuilderModal)**，管理员可预先连线“抠图 → 裂变 → 套图”节点并固化参数，发布后供全站用户复用。
- **官方套图模版**：管理不同品类（如手机壳、T恤）的商品底图、缩略图及对应的图像合成蒙版数据。

### 4.5 账号与任务管控
*对应代码：`UsersPage.tsx`, `TasksPage.tsx`*
- **前台用户管理**：手动开设、编辑、停用前台（GOGO POD 工作台）的登录账号，支持演示账号分配。
- **全站任务监控**：查阅全站正在排队、执行中、已失败或已完成的 AI 任务列表。方便在任务积压或报错时进行人工干预（重试或废弃）。

### 4.6 系统全局设置 (Settings)
*对应代码：`SettingsPage.tsx`*
- 基础网关与身份校验令牌（Admin Token）配置。
- 外部存储（OSS/S3）相关的 Bucket 与凭证占位配置，保障真实文件的稳定上传与下发。

## 5. 架构与数据持久化对接现状
- 后台 UI 当前使用 `store.ts` 内存或 `localStorage` 进行状态管理（部分处于 Mock 阶段）。
- **核心待办（A06 工程化专项）**：需要将后台的 FeatureDetail/Presets 配置面板通过 `/api/admin/v1/*` 接口与 Express Server 的 SQLite (`feature_configs`, `feature_presets` 表) 打通，消除双份配置源，实现“后台改配、服务端生效、前台调用”的闭环。
