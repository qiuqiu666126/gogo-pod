# 管理后台 · 推荐案例

## 一句话

配置前台首页「推荐玩法 / 案例」卡片：封面、文案、点进去去哪。

## 给谁用

运营、市场。

## 入口

管理后台 → **推荐案例**

---

## 页面长什么样

案例列表，每条包含：

| 字段 | 说明 |
|------|------|
| 标题 | 卡片主标题 |
| 描述 | 一两句玩法说明 |
| 封面 / 示例图 / 视频 | 视觉素材 |
| 标签 | 高转化、节日、爆款等 |
| 跳转目标 | 某作图功能 / 某工作流模板 / 外链 |
| 状态 | 草稿 / 已发布 / 已下线 |
| 排序 | 数字越小越靠前 |

---

## 按钮与交互

| 按钮 | 点击后发生什么 |
|------|----------------|
| **新建案例** | 打开表单，填上述字段 |
| **编辑** | 修改已有案例 |
| **发布** | 前台首页展示 |
| **下线** | 前台不再展示 |
| **排序** | 拖拽或填序号，决定卡片顺序 |
| **预览** | 看卡片在前台的长什么样 |

---

## 对前台的影响

| 操作 | 用户看到 |
|------|----------|
| 发布 | 首页多一张案例卡片 |
| 下线 | 卡片消失 |
| 改跳转 | 点击卡片去不同功能/模板 |
| 改排序 | 首页卡片顺序变化 |

---

## 规则

- 未发布案例仅后台可见
- 跳转目标失效时（如模板已下线）应提示运营修复
