# 管理后台 · 任务监控

## 一句话

全站任务的后台视角：谁在跑、卡在哪、为什么失败，必要时代为重试或取消。

## 给谁用

管理员、客服、运维。

## 入口

管理后台 → **任务监控**

---

## 页面长什么样

| 区域 | 内容 |
|------|------|
| 筛选栏 | 功能类型、状态、时间、用户 |
| 任务列表 | 任务 ID、用户、类型、状态、耗时、创建时间 |
| 详情抽屉/页 | 输入图、参数摘要、输出、错误栈（内部）、日志 |

---

## 按钮与交互

| 按钮 | 点击后发生什么 |
|------|----------------|
| **查看详情** | 打开完整任务信息 |
| **重试** | 用相同输入+参数重新提交（策略允许时） |
| **取消 / 废弃** | 终止排队或卡死任务 |
| **导出**（可选） | 导出筛选结果供分析 |

---

## 对前台的影响

| 后台操作 | 用户任务中心 |
|----------|--------------|
| 重试成功 | 状态变成功，出现新结果 |
| 取消 | 状态变已取消 |
| 仅查看 | 无变化 |

---

## 规则

- 重试 / 取消是否开放给客服由权限控制
- 失败原因对用户展示用人话，详情页可看技术原因（仅后台）
- 与 **概览** 联动：失败率异常时概览预警

---

## 典型排查路径

1. 用户反馈「抠图一直失败」→ 筛选该用户 + 抠图 + 失败  
2. 看详情：是否 Key 失效、图片过大、模型超时  
3. 修 **AI 功能配置** 或通知用户换图  
4. 必要时 **重试**
