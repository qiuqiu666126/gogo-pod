# 管理后台 · 概览

## 一句话

管理员一眼看平台运行是否正常，并从预警跳到对应配置页处理。

## 给谁用

运营负责人、管理员。

## 入口

管理后台登录 → 左侧 **概览**

---

## 页面长什么样

| 区域 | 典型内容 |
|------|----------|
| 数据卡片 | 用户数、今日任务量、成功率 |
| 趋势图 | 任务量 / 失败率走势（可选） |
| 预警列表 | 未配 API Key、某功能失败率高、存储异常等 |
| 快捷入口 | 跳到 AI 功能配置、任务监控等 |

---

## 按钮与交互

| 按钮 | 点击后发生什么 |
|------|----------------|
| **刷新** | 重新加载最新统计数据 |
| **去处理**（预警卡片） | 跳转到对应模块（如补 Key、看失败任务） |

---

## 规则

- 概览 **不直接改前台**，只读 + 跳转
- 预警规则由产品定义（如：24h 失败率 > X% 标红）

---

## 对前台的影响

间接：运营根据预警修配置后，前台可用性和成功率改善。
