# 前台 · 下载中心

## 一句话

查看历史下载记录，重新下载或重新打包。

## 解决什么问题

批量下载生成需要时间，用户可能关闭页面；需要可追溯的下载入口。

## 入口

左侧菜单 → **下载中心**

---

## 页面长什么样

| 列 | 内容 |
|----|------|
| 下载名称 | 如「裂变任务_20260529.zip」 |
| 来源 | 任务中心 / 我的空间 / 上架导出 |
| 状态 | 打包中 / 可下载 / 失败 / 已过期 |
| 创建时间 | — |
| 文件大小 | — |

---

## 按钮与交互

| 按钮 | 点击后发生什么 |
|------|----------------|
| **下载** | 浏览器下载该文件（状态为可下载时） |
| **重新生成** | 打包失败或过期时重新打包 |
| **删除记录** | 仅删记录，不删我的空间原文件（按产品设计） |

---

## 规则

- 打包中：**下载** 按钮禁用，显示进度或「生成中」
- 链接过期：提示 **重新生成**
- 大文件打包可能较慢，应允许用户离开页面后再回来下

---

## 与其他模块的关系

| 来源 | 关系 |
|------|------|
| 任务中心 · 下载 | 产生记录 |
| 我的空间 · 批量下载 | 产生记录 |
| 上架 · 导出刊登包 | 产生记录 |

## 后台配置影响

系统设置：存储、链接有效期、单包大小上限
